# Portfolio Guide

## 포지셔닝
웹 프론트엔드 경험이 있는 iOS 신입 개발자

## 핵심 차별화 포인트
1. 선언형 UI의 본질 이해: React/Vue.js ↔ SwiftUI/UIKit 개념 연결
2. 코드 비교를 통한 증명: 말이 아닌 코드로 실력 입증
3. T자형 역량: iOS 전문성 + 웹 기술 활용

## 타겟 메시지
웹 개발 경험이 있는 iOS 지망생은 선언형 UI 패턴을 이미 체득한 상태이므로, 학습 곡선이 50% 이상 단축됩니다.

## 웹 → iOS 지식 전환 맵
- Component 사고 → SwiftUI View 구조
- useState/useEffect → @State/@Binding
- CSS Flexbox → HStack/VStack
- Props Drilling → Environment Objects
- Responsive Design → Adaptive Layout
- API 통신 → URLSession

## Code Compare Section
포트폴리오의 핵심 차별화 요소

4가지 비교 탭:
1. 상태 관리 (useState ↔ @State)
2. 데이터 전달 (Props ↔ Parameters)
3. 리스트 렌더링 (map ↔ ForEach)
4. 조건부 렌더링 (삼항 연산자 ↔ if-else)

## 면접 준비 Q&A

### Q: "왜 iOS 개발자가 되려고 하나요?"
웹 프론트엔드를 하면서 선언형 UI의 강력함을 체감했습니다. React의 컴포넌트 기반 사고방식을 SwiftUI에 적용했을 때 학습 속도가 매우 빨랐고, iOS 앱 개발의 견고함과 사용자 경험에 매료되었습니다.

### Q: "웹 개발 경험이 iOS 개발에 어떻게 도움이 되나요?"
6가지 핵심 영역에서 직접적인 도움이 됩니다. 예를 들어 React의 useState 패턴을 이미 체득한 상태에서 SwiftUI의 @State를 배우니, 상태 관리 개념을 새로 배울 필요가 없었습니다.
